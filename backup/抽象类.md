# 抽象类

| 修饰目标     | 规则                                                         | 说明                                                         |
| :----------- | :----------------------------------------------------------- | :----------------------------------------------------------- |
| **类**       | `abstract` 修饰 → 抽象类                                     | 1、可包含0个或多个抽象方法 <br />2、不能用 `new` 实例化 <br />3、设计为被继承 |
| **方法**     | `abstract` 修饰 → 抽象方法                                   | 1、只有声明（无方法体） <br />2、必须在子类中实现（除非子类也是抽象类） |
| **相互约束** | 1、含抽象方法的类 **必须** 声明为抽象类<br />2、子类继承抽象类时，**必须** 实现所有未实现的抽象方法，否则子类也需声明为 `abstract` | 确保抽象方法最终被实现                                       |



## 总结

1. 不能new这个抽象类，只能靠子类去实现它：约束！
2. 抽象类中可以写普通的方法
3. 抽象方法必须在抽象类中

- 抽象的抽象：约束



### 思考

- 抽象类不能`new`对象，那么存在构造器么？
  - 存在构造器
- 存在的意义：
  - 提高开发效率
